## How I Think About Data Work

> [!quote] Partnership over service.
> The best analytics work happens when you're embedded in the business — sitting in pipeline calls, understanding the sales motion, knowing which metric actually drives decisions. I'd rather be in the room shaping the question than waiting for a ticket.

> [!quote] Own it to 100%.
> I don't hand off half-baked analysis. If my name is on it, I've sanity-checked the numbers, pressure-tested the edge cases, and thought about what the reader will do with it. The goal is to build enough trust that when people see your work, they act on it.

> [!quote] Build systems, not dashboards.
> A dashboard is an output. The real value is the system underneath — clean models, documented logic, reliable pipelines, and a semantic layer that lets anyone get the right answer without asking an analyst. I invest in the infrastructure so the team can scale.

> [!quote] Start simple, then iterate.
> I resist the urge to over-engineer. Ship the 80% solution, get it in front of stakeholders, and let their feedback drive the next iteration. The best models I've built started as a single SQL query in a notebook.
